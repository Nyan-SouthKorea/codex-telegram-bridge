# Agent Guide

## 목적

이 문서는 이 레포에서 작업하는 Codex/에이전트가 문서와 구현을 어떻게 유지할지 정의합니다.  
이 문서의 목표는 컨텍스트가 압축되어도 현재 구조와 규칙을 잃어버리지 않게 만드는 것입니다.

## 읽기 순서

작업 전에는 아래 순서를 지킵니다.

1. [docs/README.md](docs/README.md)
2. [docs/status.md](docs/status.md)
3. 변경 대상 코드와 직접 연결된 문서만 추가로 읽기

## 문서 운영 원칙

- `docs/status.md`는 현재 사실의 단일 기준 문서입니다.
- 문서는 append-only 로그처럼 불필요하게 덧붙이지 않습니다.
- 같은 내용이 이미 있으면 새로 추가하지 말고 기존 내용을 갱신합니다.
- 완료된 구현은 문서에도 완료 상태로 덮어씁니다.
- 서비스 경로, 설치 스크립트, 템플릿처럼 레포 안에 둘 수 있는 것은 반드시 레포 안에 둡니다.
- 레포 밖에만 존재하는 운영 사실은 `docs/status.md`에 위치와 이유를 기록합니다.
- 문서를 기준으로 사용자에게 안내할 때는 짧고 분명하게 설명하되, 친절한 말투를 사용합니다.

## 보안 원칙

- 보안 규칙의 본문은 [docs/security.md](docs/security.md)를 기준으로 봅니다.
- 이 문서에는 보안 세부 항목을 중복 복사하지 않습니다.

## 구현 원칙

- 현재 아키텍처는 `OpenClaw` 없이 `Telegram direct bot + codex-bridge + Codex CLI`입니다.
- 기능 추가가 필요하면 먼저 이 구조를 유지하는지 확인합니다.
- `Codex TUI`를 키보드 자동화로 조작하지 않습니다.
- 텔레그램 버튼/명령은 결정형 동작만 수행하고, 실제 작업은 `Codex`에게 맡깁니다.

## 변경 후 체크

코드나 운영 흐름을 바꿨다면 최소한 아래를 갱신합니다.

- [docs/status.md](docs/status.md)
- 관련 설치/운영 문서
- 필요한 경우 테스트나 스크립트

## 검증 원칙

- 가능하면 자동 테스트를 먼저 추가하거나 갱신합니다.
- 텔레그램 실제 검증을 했으면 무엇을 검증했는지 `docs/status.md`에 반영합니다.
- 실패한 가설은 문서에 장황하게 남기지 말고, 현재 유효한 사실만 남깁니다.
